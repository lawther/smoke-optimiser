# Implementation Plan: smoke-optimiser CLI

## Context

The smoke-optimiser project is a greenfield Python CLI tool and pytest plugin that analyses a test suite to produce a minimal smoke suite ŌĆö a subset delivering maximum branch coverage in minimum runtime. The PRD (`PRD-smoke-optimiser.md`) is complete; no source code exists yet. This plan covers the full CLI implementation broken into small, independently testable commits.

## Key Decisions

- **Build backend**: hatchling (PEP 517/518 compliant)
- **CLI framework**: typer (type-hint driven, built on click, `CliRunner` for testing)
- **Include/exclude filtering**: glob patterns on test node IDs **plus** `@pytest.mark.X` marker syntax (markers captured during profiling)
- **Per-test timing**: extracted from pytest's `--junit-xml` output, parsed with `xml.etree.ElementTree`
- **psutil**: optional dependency ŌĆö imported at runtime with `try/except`, fields degrade to `None` when absent
- **Pydantic**: for all external data boundaries (pyproject.toml config, coverage JSON, smoke suite file)
- **Frozen dataclasses**: for all internal data structures
- **Branch sets**: `frozenset[str]` for immutability and O(1) hashing
